## 3. Main Specification Requirements

The spec requires a true Pareto optimization framework.

The framework must maximize these objectives:

- `MSR(S)` - Mutation Score Retention
- `CGCC(S)` - Call Graph Centrality Coverage
- `ETR(S)` - Execution Time Reduction

The framework must enforce this hard constraint:

- `CR(S) >= 0.95`

The framework must return:

- a Pareto front
- not a weighted-sum single solution

The framework must use Deb-style constraint-domination:

1. A feasible solution beats an infeasible solution.
2. Among infeasible solutions, the smaller violation wins.
3. Among feasible solutions, Pareto dominance on `(MSR, CGCC, ETR)` is used.
